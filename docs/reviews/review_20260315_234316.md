## 🔴 Critical Issues

### [C-1] `total === 0` による NaN レンダリング

**ファイル**: `src/client/components/dashboard/MemberWorkloadCard.tsx:19`
**問題**: `m.total` が `0` のとき `m.done / m.total` は `NaN` になる。`Math.round(NaN)` も `NaN` のため、プログレスバーの `style={{ width: "NaN%" }}` はブラウザに無効値として無視され、バーが表示されない。表示上 "NaN%" のテキストは出ないが、バーがゼロ幅になり、スコアが正しく見えない。

```tsx
// ❌ 現在のコード
const pct = Math.round((m.done / m.total) * 100);
```

```tsx
// ✅ 修正案
const pct = m.total > 0 ? Math.round((m.done / m.total) * 100) : 0;
```

**理由**: API が `total: 0`（タスク未割当メンバー）を返す可能性は十分あり、その際バーが消えて稼働状況が正確に伝わらない。ゼロ除算に対するガードは必須。

### [M-2] スタイル情報がデータ型に混入（レイヤー違反）

**ファイル**: `src/types/dashboard.ts:16`
**問題**: `DeadlineItem` インターフェースに `badgeBg: string`（生のカラーコード `"#fef2f2"` 等）が含まれている。型定義はデータ構造を表現すべきであり、UIの色はコンポーネント層が決めるべき責務。将来 API 連携した際、BE がカラーコードを返すのは不自然。

```ts
// ❌ 現在のコード
export interface DeadlineItem {
  id: string;
  title: string;
  priority: TaskPriority;
  dueBadge: string;   // "今日" | "明日" | "3日後" ← これもリテラル型に
  badgeBg: string;    // ← UI層の責務がデータ型に漏れている
}
```

```ts
// ✅ 修正案: dueBadge を union 型にして badgeBg を型から除去
export type DueBadge = "今日" | "明日" | "3日後";

export interface DeadlineItem {
  id: string;
  title: string;
  priority: TaskPriority;
  dueBadge: DueBadge;
  // badgeBg は削除 → コンポーネントで DUE_BADGE_BG 定数にマッピング
}

// DeadlineCard.tsx 側
const DUE_BADGE_BG: Record<DueBadge, string> = {
  "今日": "#fef2f2",
  "明日": "#fff7ed",
  "3日後": "#f4f4f5",
};
```

**理由**: API 連携時、BE は「今日」「明日」というラベルは返せても `#fef2f2` は返さない。色のマッピングはコンポーネント側に持つのが正しいレイヤー分離。
