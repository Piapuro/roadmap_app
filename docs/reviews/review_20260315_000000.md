## 🔴 Critical Issues

### [C-1] Object URL のメモリリーク

**ファイル**: `src/client/components/profile/ProfileForm.tsx:99`
**問題**: `URL.createObjectURL()` で生成した Blob URL を `URL.revokeObjectURL()` で解放していない。ユーザーが複数回画像を選択するたびにメモリが解放されず、コンポーネントアンマウント時にも残り続ける。

```tsx
// ❌ 現在のコード
function handleFileChange(e: React.ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0]
  if (!file) return
  const url = URL.createObjectURL(file)
  setForm(f => ({ ...f, avatarPreviewUrl: url }))
}
```

```tsx
// ✅ 修正案: 古いURLを破棄してから新しいURLをセット
function handleFileChange(e: React.ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0]
  if (!file) return
  // 以前のBlob URLを解放
  if (form.avatarPreviewUrl?.startsWith('blob:')) {
    URL.revokeObjectURL(form.avatarPreviewUrl)
  }
  const url = URL.createObjectURL(file)
  setForm(f => ({ ...f, avatarPreviewUrl: url }))
}

// + コンポーネントアンマウント時にも解放
useEffect(() => {
  return () => {
    if (form.avatarPreviewUrl?.startsWith('blob:')) {
      URL.revokeObjectURL(form.avatarPreviewUrl)
    }
  }
}, [form.avatarPreviewUrl])
```

**理由**: ブラウザのメモリを徐々に消費し、長時間使用すると画像選択のたびにリークが蓄積する。

**ファイル**: `src/client/components/profile/ProfileForm.tsx:338`


```tsx

// タグの削除ボタンに onMouseDown で blur をキャンセルする処理を追加
<button
```

### [M-2] `addExpertSkill` / `addLearnSkill` のDRY違反

**ファイル**: `src/client/components/profile/ProfileForm.tsx:104–128`
**問題**: 2つの関数がほぼ同一のロジックで、状態変数名だけが異なる。

```tsx
// ❌ 現在のコード: 重複したロジック
function addExpertSkill() {
  const val = expertInput.trim()
  if (val && !form.expertSkills.includes(val)) {
    setForm(f => ({ ...f, expertSkills: [...f.expertSkills, val] }))
  }
  setExpertInput('')
  setIsAddingExpert(false)
}

function addLearnSkill() {
  const val = learnInput.trim()
  if (val && !form.learningSkills.includes(val)) {
    setForm(f => ({ ...f, learningSkills: [...f.learningSkills, val] }))
  }
  setLearnInput('')
  setIsAddingLearn(false)
}
```

```tsx
// ✅ 修正案: 汎用ハンドラに統合
function addSkill(type: 'expert' | 'learn') {
  const val = (type === 'expert' ? expertInput : learnInput).trim()
  const key = type === 'expert' ? 'expertSkills' : 'learningSkills'
  if (val && !form[key].includes(val)) {
    setForm(f => ({ ...f, [key]: [...f[key], val] }))
  }
  if (type === 'expert') { setExpertInput(''); setIsAddingExpert(false) }
  else { setLearnInput(''); setIsAddingLearn(false) }
}
```

**理由**: スキル種別が増えた場合に保守コストが増大する。

```ts
// ✅ 修正案: 明示的なフィールドマッピング
const data = await res.json()
if (!data?.id) throw new Error('Invalid response from API')
return {
  id:             data.id,
  name:           data.name,
  email:          data.email,
  bio:            data.bio ?? '',
  avatarUrl:      data.avatar_url ?? null,
  expertSkills:   data.expert_skills ?? [],
  learningSkills: data.learning_skills ?? [],
} satisfies UserProfile
```

**理由**: APIのフィールド名とTS型のプロパティ名の不一致により、ランタイムで `undefined` が混入するリスクがある。
